# NanoClaw — Summary

Personal Claude assistant built as a single Node.js orchestrator (`src/index.ts`) with a skill-based channel system. Channels (WhatsApp, Telegram, Slack, Discord, Gmail, Protonmail IMAP) self-register at startup and route inbound messages to a Claude Agent SDK instance running inside a per-group container (Linux VM) with an isolated filesystem and memory at `groups/{name}/`. The OneCLI gateway (port 3002) injects secrets at request time so containers never see raw keys. Scheduled tasks run via `src/task-scheduler.ts`; IPC between host and containers flows through `src/ipc.ts`. Active Jeff-specific deployment: Telegram + Gmail + Protonmail channels registered, agent persona "Madison", persistent data under `~/containers/data/NanoClaw` (BTRFS subvolume, auto-snapshotted; see `lode/infrastructure/persistence.md`), downloads to `/workspace/downloads` inside containers.
